Expected Format

The expected format for the different problem types can be seen as follows:

Note: It is assumed you will apply logically appropriate parentheses on your statements. Order of precedence will still be applied if not used.

Example: AvB ✅

Example: Av(B^C) ✅

Example: AvB^C ❌

Logical Status

The logical status will appear as a selectable field when only the standard operators are used (^, v, >, ~)

Example

Equivalence

The equivalence will appear as a selectable field when a double colon (s1 :: s2) is used to separate two statements

Example

Consistency

The consistency will appear as a selectable field when a comma (s1, s2, … , sx) is used to separate two or more statements

Example

Validity

The validity will appear as a selectable field when a colon followed by a period (p1, p2, …, px :. c1) is used to separate two or more premises from a conclusion statement
